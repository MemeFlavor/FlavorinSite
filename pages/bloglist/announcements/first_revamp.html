Sidebar

Anyways I'm gonna start with the sidebar redesign because I have a lot to talk about this. This new design is a bit inspired by this website's sidebar design with it's news channel asthetic. The website is basically an ARG from what I can tell, I just randomly discovered this website while websurfing for any design inspiration. So I took some elements I like and implemented it. You can obviously tell with the headbar having a CRT effect, the title (at the left) having a special background, and the subtitle (at the right) having a blinking background animation within the Sidebar text.

About the CRT effect, I didn't make it, I stole it from this website it basically explains in detailed how the effect is made. I only grabbed the screen-door and flicker effects and just implemented into SCSS. But I did some minor modifications—since it's SCSS I decided to randomize the opacity values of the keyframe percentages within the flickering animation. To add a more authentic feeling of randomness to the effect, and totally not flexing my skills (trust me). 

$flicker-proximate: 100000;
@keyframes flicker {
     @for $flicker-index from 0 through 20 {
     $flicker-sections: math.percentage(math.div(5 * $flicker-index, 100));
     $flicker-vars: math.div(math.random($flicker-proximate), $flicker-proximate);

     #{$flicker-sections} {
          opacity: $flicker-vars;
     }
}

This is the code that I've implemented, it basically loops from 0 through 20 within the keyframe animation. Because since the original CSS code has every percentage marker being intervals in 5, I obviously multiply it with the $flicker-index for-loop variable. Dividing it by 100 and converting the result in the math.percentage() method, all happening within the $flicker-sections variable.

For the randomize values, I used the math.random() method and with the limit being 100,000k and dividing it by 100,000k in order to have hundred-thousandths place for the opacity values, at the $flicker-vars variables.

Finally applying those variables to their respective places, the $flicker-sections is placed at the marker, interpolated converting it into a real percentage marker. And the $flicker-vars is at the opacity values. Now with all of that explanation here's the converted code, to try and give you an idea:

Another thing you can see from my inspirations is the marquee behind the title in each sections of the sidebar. And it's actually easy to implement the overlapping of elements by just using the grid layout. 

<hgroup class="navigation__headbar">
     <h3 class="navigation__headbar-title">Navigation</h3>
     <marquee class="navigation__headbar-marquee">Explore! • Explore! • Explore!</marquee>
</hgroup>

Here we have an example on how implemented it, this HTML code has an element hgroup containing two child elements which will be overlapping eachother. 

.navigation__headbar {
     display: grid;
     grid-template-columns: 1fr;
     grid-template-rows: 1fr;
     align-items: center; // added this visualize the overlapping by making the text center

     & > * { // Supported now
          grid-area: 1 / 1;
     }
}

.navigation__headbar-title {
     z-index: 1;
     /* Insert CSS Stuff */
}

.navigation__headbar-marquee {
     /* Insert CSS Stuff */
}
This code block will be updated with a newer design

In CSS, convert the parent element, such as this hgroup, to a grid layout with both of its template columns and rows set to 1fr. Set the grid area of each child element to 1 / 1 to make the start and end of each column and row, set to 1. Finally choose which child element's order by setting its z-index property. Boom your done!

And also to clarify the nesting, it's actually supported to all browsers since 2023. Important to note that it is extremely different to SCSS nesting, but I'm way too lazy to explain it right now. You can read this blog post I found, if you have the time. Also another thing try not to overnest your CSS code—enforce the three-level limit to make your code maintainable. Making it easier to read and understand, also makes it less uglier to look at.
