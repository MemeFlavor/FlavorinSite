But I did some minor modifications—since it's SCSS I decided to randomize the opacity values of the keyframe percentages within the flickering animation. To add a more authentic feeling of randomness to the effect, and totally not flexing my skills (trust me).

Flavor Code SCSS
99 
@use "sass:math";

$flicker-proximate: 100000;
@keyframes flicker {
     @for $flicker-index from 0 through 20 {
          $flicker-sections: math.percentage(math.div(5 * $flicker-index, 100));
          $flicker-vars: math.div(math.random($flicker-proximate), $flicker-proximate);

          #{$flicker-sections} {
               opacity: $flicker-vars;
          }
     }
}

This is the code that I've implemented, it basically loops from 0 through 20 within the keyframe animation. Because since the original CSS code has every percentage marker being intervals in 5, I obviously multiply it with the $flicker-index for-loop variable. Dividing it by 100 and converting the result in the math.percentage() method, all happening within the $flicker-sections variable.

For the randomize values, I used the math.random() method and with the limit being 100,000k and dividing it by 100,000k in order to have hundred-thousandths place for the opacity values, at the $flicker-vars variables.

Finally applying those variables to their respective places, the $flicker-sections is placed at the marker, interpolated converting it into a real percentage marker. And the $flicker-vars is at the opacity values. Now with all of that explanation here's the converted code, to try and give you an idea:

Another thing you can see from my inspirations is the marquee behind the title in each sections of the sidebar. And it's actually easy to implement the overlapping of elements by just using the grid layout.

Flavor Code HTML
99 
<hgroup class="navigation__headbar">
     <h3 class="navigation__headbar-title">Navigation</h3>
     <marquee class="navigation__headbar-marquee">Explore! • Explore! • Explore!</marquee>
</hgroup>

Here we have an example on how implemented it, this HTML code has an element hgroup containing two child elements which will be overlapping eachother.

Flavor Code CSS
99 
.navigation__headbar {
     display: grid;
     grid-template-columns: 1fr;
     grid-template-rows: 1fr;
     align-items: center; // added this to visualize the overlapping by making the text center

     & > * {
          grid-area: 1 / 1;
     }
}

.navigation__headbar-title {
     z-index: 1;
     /* Insert CSS Stuff */
}

.navigation__headbar-marquee {
     /* Insert CSS Stuff */
}

In CSS, convert the parent element, such as this hgroup, to a grid layout with both of its template columns and rows set to 1fr. Set the grid area of each child element to 1 / 1 to make the start and end of each column and row, set to 1. Finally choose which child element's order by setting its z-index property. Boom your done!

And also to clarify the nesting, it's actually supported to all browsers since 2023. Important to note that it is extremely different to SCSS nesting, but I'm way too lazy to explain it right now. You can read this blog post I found, if you have the time. Also another thing try not to overnest your CSS code—enforce the three-level limit to make your code maintainable. Making it easier to read and understand, also makes it less uglier to look at.


For the navigation list, it has significantly improved greatly. Each item now only has one element, previously it contains another element which basically represent the side block decoration attached to the item. Now it uses a psuedo-element ::before to represent the side block. It basically just injects an element that is the first child of the selected element.

Flavor Code CSS
99 
.item {
     position: relative;

     &::before {
          content: "";
          position: absolute;
          top: 0;
          width: 8px;
          height: calc(1rem + 9px);
     }
}

The way I did was pretty simple, you add the content property to the psuedo-element so that the element can be displayed, this is important. I made the element have its position set to relative. The reason being for the psuedo-element positioned relative to the nearest positioned ancestor, since it's position value is set to absolute. Alongside setting its position properties (i.e top, bottom, etc) with their given values and adding the width and height properties respectively.

Amy

Remember how I added Amy randomly dancing in each direction at the home page. I'm gonna explain how she works with her whole dancing movement thingy, I'm promising you this is one of the most easier script to understand for dummies like you. And for the best part you can use this in your very own website, with or without credit! Tho a bit of credit will be nice tho. Anyways here's the script for Amy:

Flavor Code Javascript
99 
function getRandomInt(min, max) {
     return Math.floor(Math.random() * (max - min + 1)) + min;
}

const LIMIT = 60
const STEPS = 10
const amy = document.querySelector('.music__album-amy')

let position = 0
const intervalId = setInterval(() => {
     let positionRancrement = getRandomInt(-1, 1)
     if (positionRancrement != 0) {
          position += positionRancrement * STEPS
     }

     if (position >= -LIMIT && position <= LIMIT) {
          amy.style.transform = `translateX(${position}px)`
     } else if (position <= -LIMIT || position >= LIMIT) {
          position -= positionRancrement * STEPS
     }
}, 2500);

Let's focus on the variables first, specifically the variables at the top of the script.

  • At line 5, the LIMIT variable tells the allowed range will Amy move throughout the script.
  • At line 6, the STEPS variable tells how much the step amount will Amy move throughout the script.
  • At line 7, the amy variable just gets the element where Amy is at.
  • At line 9, the position variable is the current position that Amy is in.

Now for the main part of the script which is the dancingMovement variable which holds a setInterval() function, that basically executes the given code at a fixed time interval. It contains two arguments, which are literal values that you passed towards a function. An arrow function that basically contains the main code and the delay time in which the code will execute in intervals. Which is 2500 milliseconds (2.5 seconds).

The arrow function (the main code), we've got the positionRancrement variable that randomizes an integer value from -1 to 1. Which will be multiplied by the STEPS variable then added and assign to the position variable, determining which direction she'll go by its value added. It will only do this, if the positionRancrement variable is not 0; preventing a multiplication of 0.

At the last section we have if statement chain, it first checks whether Amy is within an acceptable range for movement. If it is she can randomly move in any direction. Otherwise she will attempt to move in the opposite direction to a acceptable range for movement. To prevent her overflowing on other elements on the page.
